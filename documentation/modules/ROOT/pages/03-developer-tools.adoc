= Lab: Automation Developer Experience with DevSpaces

[abstract]
This lab walks through the various aspects of launching, configuring and using DevSpaces on OpenShift as your primary development environment for Ansible automation development.

== 1. Introduction: Why DevSpaces for Ansible Development?

DevSpaces provides a cloud-based, containerized development environment that eliminates local setup complexity and ensures consistency across development teams.

.Why DevSpaces for Ansible?

. **Pre-configured Environment**: All Ansible tools and dependencies pre-installed
. **Collaborative**: Share workspaces and configurations with team members
. **Scalable**: Access powerful development resources on-demand
. **Integrated**: Built-in terminal, editor, and version control
. **Consistent**: Same environment for all developers regardless of local setup

== 2. Lab Setup: Accessing Your DevSpaces Environment

Before starting, ensure you have access to your OpenShift DevSpaces environment.

=== Step 2.1: Navigate to DevSpaces Console

. Log in to your OpenShift cluster web console
. Navigate to the Developer perspective
. Click on "DevSpaces" in the left navigation menu
. This will open the DevSpaces dashboard

=== Step 2.2: Understanding the DevSpaces Interface

The DevSpaces console provides:

. **Workspaces**: Your development environments
. **Factories**: URL-based workspace creation
. **Settings**: User preferences and configurations

== 9. Configuring Ansible Navigator

=== Step 9.1: Ansible Navigator Setup

[source,bash,role=execute]
----
# Check ansible-navigator installation
ansible-navigator --version

# Create ansible-navigator configuration
cat > ansible-navigator.yml << EOF
---
ansible-navigator:
  execution-environment:
    image: registry.redhat.io/ubi9/ubi-init:latest
  playbook-artifact:
    enable: false
EOF
----

=== Step 9.2: Understanding ansible-navigator

Ansible Navigator provides:

. **Interactive execution**: Step through playbook execution
. **Execution environments**: Isolated Ansible execution
. **Rich output**: Enhanced display of results

== 10. Running Playbooks with Ansible Navigator

=== Step 10.1: Basic Playbook Execution

[source,bash,role=execute]
----
# Run a playbook with ansible-navigator
ansible-navigator run playbook.yml --mode interactive
----

=== Step 10.2: Using Execution Environments

[source,bash,role=execute]
----
# Specify execution environment
ansible-navigator run playbook.yml \
  --execution-environment-image registry.redhat.io/ubi9/ubi-init:latest
----
